# 第 1 周术语表

更新时间：2026-04-14

这份术语表只收第 1 周必须会说清楚的词，不追求大而全。

## Prompt

你发给模型的输入整体，包括指令、上下文、样例、格式要求等。

要点：

- prompt 不只是“那一句问题”
- 多轮对话里，历史消息也会成为 prompt 的一部分

## System prompt

传统说法，指系统级高优先级指令，用来设定助手角色、边界、规则。

补充：

- 在新一代 OpenAI API 文档里，更常见的是 `developer` 消息
- 学习时可以先把它理解为“更现代的 system prompt 表达方式”

## Developer message / Developer instruction

应用开发者给模型的高优先级规则，通常高于 `user` 消息。

典型内容：

- 身份设定
- 输出格式
- 工具调用规则
- 安全边界

## User message

终端用户当前这次任务的输入。它告诉模型“我这次要你做什么”。

## Zero-shot

不给样例，只给任务说明，让模型直接完成任务。

适用：

- 任务简单
- 输出要求清楚
- 你想先测最小可行 prompt

## One-shot

给 1 个样例，再让模型做类似任务。

适用：

- 你只需要固定一个很明显的输出模式

## Few-shot

给少量输入/输出样例，让模型学会你想要的任务模式。

适用：

- 分类标签固定
- 抽取格式固定
- 风格要求明确

风险：

- 例子太多会涨成本
- 例子不一致会误导模型

## Token

模型处理文本时的基本分块，不等于“字”也不等于“词”。

实用理解：

- 输入越长，token 越多
- 输出越长，token 也越多
- 成本通常按 token 计

## Cost

一次模型调用的费用。常见组成包括：

- input tokens
- cached input tokens
- output tokens
- tool calls

## Context window

模型一次调用能处理的总上下文预算。

关键点：

- prompt 和输出都占用预算
- 历史消息、文档内容、工具返回结果也会占用预算
- 超过窗口后，系统必须截断、压缩，或者报错

## Temperature

控制输出随机性和发散度的参数。

粗略理解：

- 低 temperature：更稳定，更保守
- 高 temperature：更发散，更有变化

注意：

- 它不是“质量旋钮”
- 不是所有模型或场景都适合频繁调它

## Schema

对输入或输出结构的明确约束，常见为 JSON Schema、Pydantic 模型或函数参数签名。

价值：

- 帮你把“格式要求”说清楚
- 降低输出飘掉的概率
- 方便程序校验和解析

## Structured output

要求模型按指定结构输出，而不是自由文本输出。

常见形式：

- JSON
- 表格
- 严格字段列表

## Retry

当请求失败、格式不合法、工具报错时，再试一次的机制。

常见原因：

- 网络或 API 临时失败
- 输出不符合 schema
- 工具参数缺失

注意：

- retry 不是无限重试
- 要配合最大次数、超时和错误分类

## Latency

一次请求从发出到拿到结果的耗时。

常被影响的因素：

- 模型类型
- prompt 长度
- 输出长度
- 工具调用

## Prompt caching

把重复使用的 prompt 前缀缓存起来，减少延迟和成本。

实用含义：

- 固定不变的说明、规则、模板，尽量放前面
- 这样更容易命中缓存

## Tool

模型可以调用的外部能力，比如查网页、查数据库、执行函数。

重点：

- tool 是系统暴露给模型的能力
- 不是模型“凭空知道”怎么做

## Workflow

为完成任务而设计的一组稳定步骤。

例子：

- 先抽取字段，再校验格式，再人工审核

## Guardrails

限制系统行为的保护措施。

例子：

- 工具白名单
- 参数校验
- 超时
- max steps
- 人工确认

## Max output tokens

一次响应允许生成的最大输出 token 数。

作用：

- 控制回答长度
- 控制成本
- 防止模型过度展开

## Context truncation

当上下文太长时，系统丢弃或压缩一部分内容。

为什么危险：

- 模型可能失去最关键的前文
- 结果会变得不稳定，甚至答非所问

## 本周最低掌握标准

你至少要能用自己的话说明下面 8 个词：

- `developer instruction`
- few-shot
- token
- cost
- context window
- schema
- retry
- workflow
