# AI Agent / 大模型学习工作区

这套仓库定制了我的 24 周学习路线，落成了一个可执行的学习系统，目标不是“看过很多概念”，而是持续产出项目、实验、复盘和作品集。

## 先看什么

1. 看 [docs/01-roadmap.md](/D:/Learning/docs/01-roadmap.md) 理解 24 周主线。
2. 看 [docs/02-projects-and-checkpoints.md](/D:/Learning/docs/02-projects-and-checkpoints.md) 明确每阶段要交付什么。
3. 看 [docs/04-weekly-operating-system.md](/D:/Learning/docs/04-weekly-operating-system.md) 确定你每周怎么推进。
4. 从 [templates/daily-log.md](/D:/Learning/templates/daily-log.md) 和 [templates/weekly-review.md](/D:/Learning/templates/weekly-review.md) 开始记录。

## 仓库结构

```text
docs/          路线图、项目要求、资源清单、每周推进方法
templates/     学习日志、实验报告、周复盘模板
notes/         你的阅读笔记、术语解释、框架比较
experiments/   检索、prompt、agent 版本对比实验
projects/      每阶段项目说明与实现目录
reports/       实验报告、阶段总结、公开作品文稿
```

## 你的默认节奏

- 每周 6-8 小时
- 以 Python 为主线
- 前 12 周聚焦：LLM 基础、tool calling、单 agent、RAG、eval、MCP
- React 放在产品化阶段，不抢前排
- multi-agent、DSPy、computer use 作为增强项，不作为入门主线

## 本仓库的使用规则

- 每学完一个主题，至少留下一份可检索产物：笔记、代码、实验或复盘。
- 每完成一个项目，必须补一份失败分析，不能只写“跑通了”。
- 每做一个重要判断，优先引用官方文档、benchmark 或代码仓库。
- 如果一个方向无法回答“什么时候该用、什么时候不该用”，说明还没学透。

## 建议起步顺序

- 第 1 周：读 `01-roadmap`，建自己的学习台账
- 第 1-2 周：完成结构化抽取器
- 第 3-4 周：完成工具型 FAQ 助手
- 第 5 周起：开始把每周结果沉淀到 `notes/`、`experiments/`、`reports/`
